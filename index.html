Why multiparametric optimization helps

Different starts need different best drives.

Solving the problem again for every new starting position can be slow. Multiparametric optimization calculates the answers in advance.

Calculate first. Use the answer right away.

It groups starting positions that use the same type of best drive. It then stores the right control rule for each group.

Measure the starting position → Find its group
(critical region) → Use the stored control rule
This demo recalculates the answer when you move the slider so you can see the change. A full multiparametric controller would calculate all groups and rules before the car starts.

How the best drive is calculated

The computer chooses the sideways acceleration u(t) over the 8-second drive.

Objective
Minimize  J = ∫08 u(t)2 dt

Use as little strong sideways movement as possible.

Car model
ẏ(t) = v(t),    v̇(t) = u(t)

y is sideways position, v is sideways speed, and u is sideways acceleration.

Starting condition
y(0) = y0,    v(0) = 0

The car starts at the selected position with no sideways movement.

Finish condition
y(8) = 0,    v(8) = 0

Finish in the center and stop moving sideways.

Gate conditions
Gate 1 at 2.0 s: −2.0 ≤ y(2.0) ≤ −0.8 m
Gate 2 at 4.1 s:  1.1 ≤ y(4.1) ≤  2.3 m
Gate 3 at 6.2 s: −1.8 ≤ y(6.2) ≤ −0.6 m

The car must be inside each glowing gate when it reaches that gate.

Gold car: among all drives that satisfy the model and every condition above, the gold drive has the smallest value of J. It is calculated, not drawn by hand.

This simple model is used to make the control idea easy to see.
